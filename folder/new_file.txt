1. Project Structure Scenarios
These happen because real repos are messy.
1.1 Modules with side effects on import
Examples:
- DB connections
- API calls
- reading files
- environment variable dependencies
Your agent must sandbox imports or mock them.

1.2 Modules that require configuration
Tests fail because:
- missing .env
- missing config files
- missing secrets
Your agent must detect and neutralize config dependencies.

1.3 Modules that import optional dependencies
Example:
try:
    import numpy as np
except ImportError:
    np = None


Your agent must generate tests that handle both branches.

1.4 Modules with circular imports
LLM‑generated tests often trigger circular import failures.
Your agent must detect and avoid them.

 2. Test Generation Scenarios
These are LLM‑specific failure modes.
2.1 LLM hallucinating nonexistent functions
Even with signatures, the model may invent:
- helper functions
- constants
- classes
- parameters
Your refinement loop must detect and eliminate hallucinations.

2.2 LLM generating tests that mutate global state
Example:
GLOBAL_CACHE.append("x")


This causes nondeterministic failures across iterations.

2.3 LLM generating tests that depend on execution order
Example:
test_b assumes test_a created a file


Your agent must enforce test isolation.

2.4 LLM generating tests that rely on randomness
Example:
assert random.randint(1, 10) == 5


Your agent must detect and remove nondeterminism.

3. Pytest Runtime Scenarios
These are failures that only appear when pytest actually runs.
3.1 Tests that hang or never complete
Common causes:
- infinite loops
- waiting for network
- waiting for input()
Your agent must enforce timeouts.

3.2 Tests that crash pytest itself
Examples:
- segmentation faults via C extensions
- recursion depth exceeded
- memory exhaustion
Your agent must detect catastrophic failures.

3.3 Tests that interfere with each other
Examples:
- modifying working directory
- modifying environment variables
- modifying global state
Your agent must detect cross‑test contamination.

 4. Coverage Scenarios
Coverage introduces its own set of problems.
4.1 Modules with unreachable code
Example:
if False:
    ...


Your agent must not try to test unreachable branches.

4.2 Modules with dynamic behavior
Example:
def f():
    return globals()


Coverage tools may misreport these.

4.3 Modules that import large libraries
Coverage slows down dramatically.
Your agent must detect and skip heavy modules.

 5. Refinement Loop Scenarios
These are meta‑failures in the refinement engine itself.
5.1 Oscillation
LLM alternates between two wrong versions:
- iteration 1: fix A, break B
- iteration 2: fix B, break A
Your agent must detect oscillation and break the loop.

5.2 Convergence failure
Coverage never reaches threshold because:
- module is too complex
- LLM keeps hallucinating
- missing functions are untestable
Your agent must detect “unreachable coverage” and stop.

5.3 Prompt drift
Over many iterations, the LLM:
- adds noise
- changes style
- introduces new errors
Your agent must enforce deterministic formatting.

5.4 Signature mismatch detector failing on complex AST
Examples:
- decorators
- nested functions
- lambdas
- partials
Your signature detector must be hardened.

 6. Repository‑Level Scenarios
These appear only when running on real repos.
6.1 Modules that should not be tested
Examples:
- __init__.py
- migration scripts
- CLI entrypoints
- data files
Your agent must skip them.

6.2 Tests that require external services
Examples:
- Redis
- Postgres
- AWS
- Kafka
Your agent must detect and mock external dependencies.

6.3 Modules with side‑effect imports
Example:
import pandas as pd
pd.read_csv("missing.csv")


Your agent must detect import‑time side effects.

 7. LLM Integration Scenarios
These are operational issues.
7.1 LLM returning partial output
Your agent must detect incomplete test files.

7.2 LLM returning non‑Python output
Examples:
- markdown
- explanations
- JSON
Your agent must sanitize output.

7.3 LLM rate limits or timeouts
Your agent must retry gracefully.

 8. Human‑Code Scenarios
Real developers write messy code.
8.1 Functions with misleading names
Example:
def add(a, b):
    return a - b


Your agent must infer behavior from source, not name.

8.2 Functions with side effects
Example:
def f(x):
    print(x)


Your agent must capture stdout.

8.3 Functions that depend on system time
Example:
return datetime.now()


Your agent must mock or freeze time.

 9. Security Scenarios
Tests may accidentally:
- read sensitive files
- write to disk
- send network requests
- modify environment variables
Your agent must sandbox execution.

 10. Scaling Scenarios
When running on large repos:
- thousands of modules
- hundreds of thousands of lines
- long refinement cycles
- memory pressure
- slow pytest runs
Your agent must optimize:
- caching
- parallelization
- incremental refinement
